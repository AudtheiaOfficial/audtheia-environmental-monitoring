Open-source · FAIR data · Marine + Terrestrial

Environmental monitoring at the intersection of computer vision and multi-agent AI

Audtheia is a distributed pipeline integrating real-time Porifera detection on RTSP video streams, nine-agent post-detection ecological analysis, 72-point structured observation records following FAIR data principles, and automated daily PDF intelligence reports — designed for deployment in marine protected areas and terrestrial field research contexts.

60 FPS RTSP inference
72 data points per observation

System architecture

A four-platform detection and analysis pipeline

Data originates at the camera and flows through detection, multi-agent ecological analysis, structured storage, and automated reporting without manual intervention at any stage. Each platform operates independently and communicates through defined API boundaries.

INPUT RTSP Stream live video · 60 FPS
→
01 / Roboflow Detection YOLOv11 · classifier
→
02 / N8N Analysis 9 agents · GPT-4o
→
03 / Airtable Storage 72 fields · REST

OUTPUT Reports PDF · dashboard
01

Roboflow

official-porifera-classifier-ju8er/12

Custom-trained Porifera classifier and YOLOv11 object detection model, running inside a Roboflow Workflow via a custom Python block. Claude 3.5 Sonnet (Anthropic API) is integrated at the workflow level for real-time RTSP inference at 60 FPS.

  • Hosted inference endpoint at detect.roboflow.com
  • Hosted Image and Hosted Video inference both available
  • Anthropic API key encrypted within Roboflow configuration — never transmitted to any client
YOLOv11 Claude 3.5 Sonnet RTSP
02

N8N

RTSP Analyst Workflow · Daily Reporter Workflow

Nine specialized AI agents perform deep post-detection ecological analysis using GPT-4o (RTSP Analyst Workflow). Five additional agents generate automated daily PDF intelligence reports via API Template, incorporating satellite imagery and multi-database cross-references.

  • 9 agents — RTSP Analyst Workflow (ecological analysis)
  • 5 agents — Daily Reporter Workflow (automated PDF generation)
  • OpenAI API key isolated within N8N credential system — never touches the browser
GPT-4o Multi-agent Automation
03

Airtable

api.airtable.com · REST · browser CORS

Structured research database with two primary observation tables. The REST API at api.airtable.com supports direct browser CORS calls — the dashboard reads observational data without a proxy layer, keeping credentials in browser local storage only.

  • Species Observations — 62 columns
  • Environmental Mapping — 10 columns
  • 72 data points per complete observation record
FAIR data REST API Structured

Integrations

14+ scientific data sources

Each detection is automatically cross-referenced against established biodiversity repositories and environmental monitoring services, situating every observation within the broader ecological and geospatial record.

  • GBIF · iNaturalist · IUCN · NOAA · Open-Meteo
  • Seven additional scientific and environmental databases
  • Mapbox Static Images API — satellite imagery at 10×, 13×, 15× zoom
GBIF IUCN Mapbox
Designed for institutional research

Built for the workflows of marine and environmental science institutions

Audtheia operates on a bring-your-own-account model. Institutional researchers maintain complete ownership of their observational data within their own Roboflow, N8N, and Airtable environments — no data passes through Audtheia infrastructure. All required platforms offer free tiers sufficient for field research deployment.
