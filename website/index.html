Open-source · FAIR data · Marine + Terrestrial

Environmental monitoring at the intersection of computer vision and multi-agent AI

Audtheia is a distributed pipeline integrating real-time Porifera detection on RTSP video streams, nine-agent post-detection ecological analysis, 72-point structured observation records following FAIR data principles, and automated daily PDF intelligence reports, designed for deployment in marine protected areas and terrestrial field research contexts.

System architecture

A four-platform detection and analysis pipeline

Data originates at the camera and flows through detection, multi-agent ecological analysis, structured storage, and automated reporting without manual intervention at any stage. Each platform operates independently and communicates through defined API boundaries.

01

Roboflow

official-porifera-classifier-ju8er/12

Audtheia's detection layer runs inside a Roboflow Workflow using a custom Python block. The official Porifera classifier (official-porifera-classifier-ju8er/12) was trained using RF-DETR (Medium) and serves as the reference implementation. Users deploy their own custom-trained classifiers and detection models to match their target species and field conditions.

  • Hosted inference endpoint at detect.roboflow.com
  • Hosted Image and Hosted Video inference both available
  • Custom Python block enables arbitrary workflow logic and third-party API integration at the inference step

02

N8N

RTSP Analyst Workflow · Daily Reporter Workflow

Two independent N8N workflows handle post-detection processing. The RTSP Analyst Workflow runs nine specialized agents for deep ecological analysis, cross-referencing detections against biodiversity and environmental databases. The Daily Reporter Workflow runs five agents to compile and export structured PDF intelligence reports.

  • 9 agents — RTSP Analyst Workflow (ecological analysis)
  • 5 agents — Daily Reporter Workflow (automated PDF generation)
  • All LLM and API credentials are isolated within N8N's credential system and never transmitted to any client
Multi-agent Automation REST
03

Airtable

api.airtable.com · REST · browser CORS

Structured research database with three observation tables. The REST API at api.airtable.com supports direct browser CORS calls, so the dashboard reads observational data without a proxy layer, keeping credentials in browser local storage only.

  • Species Observations — 62 columns
  • Environmental Mapping — 10 columns
  • Daily Reports — 3 columns (finalized PDF records from the Daily Reporter Workflow)
  • 72 data points per complete observation record

Designed for institutional research

Built for the workflows of marine and environmental science institutions

Audtheia operates on a bring-your-own-account model. Institutional researchers maintain complete ownership of their observational data within their own Roboflow, N8N, and Airtable environments, with no data passing through Audtheia infrastructure. All required platforms offer free tiers sufficient for field research deployment. Audtheia is suitable for deployment by any research institution, university laboratory, or independent field researcher with access to compatible camera infrastructure.
