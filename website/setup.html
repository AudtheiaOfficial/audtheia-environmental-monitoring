Deployment Guide

Deploy Your Own Audtheia Instance

This guide covers every step required to configure a fully operational Audtheia environmental monitoring deployment. It is written for researchers who want to process their own RTSP video streams, store species observations in their own Airtable base, and view results through the Audtheia researcher dashboard.

01 Roboflow Species detection from RTSP video at 60 FPS
02 N8N RTSP Analyst 9 AI agents generating 72 data points per observation
03 N8N Daily Reporter 5 AI agents generating scheduled PDF intelligence reports
04 Airtable Research database storing all observations and reports
05 Dashboard Researcher interface connected to your Airtable base

Prerequisites

Complete the following before beginning the platform-by-platform setup. All required platforms offer free tiers sufficient for research deployments.

Hardware and access

A camera with RTSP output, or any video source compatible with Roboflow Workflows (MP4 file, webcam, IP camera stream)
GPS coordinates for your study site, either transmitted by the camera or available to enter manually in N8N
Git installed locally, with the Audtheia repository cloned (see below)

Clone the repository

Workflow templates, Airtable schemas, and the Roboflow workflow file are distributed through the repository. Clone it before proceeding with any section.

BASH 
git clone https://github.com/AudtheiaOfficial/audtheia-environmental-monitoring.git
cd audtheia-environmental-monitoring

01

Roboflow

Species detection model and real-time RTSP video processing pipeline

Roboflow hosts your species detection model and runs the RTSP Analyst pipeline that processes your video stream. Detections are generated at 60 FPS and forwarded to N8N via webhook. The workflow template contains four pre-configured custom Python blocks: Detection_Converter, Add_Webcam_Interface, Anthropic_Environmental_Analyzer, and Analyst_Caller. The Anthropic_Environmental_Analyzer block calls Claude 3.5 Sonnet for preliminary environmental analysis, and the Analyst_Caller block dispatches detection data to N8N. All four blocks require only API keys or configuration values to activate.

1

Create a Roboflow account

Sign up at roboflow.com. The free tier supports all Audtheia functionality.

2

Choose a detection model

Select the model that fits your target ecosystem and species. Three options are available:

OPTION A
Porifera Classifier (marine sponge research)

Fork workspace official-porifera-classifier-ju8er on Roboflow Universe. This is the reference implementation classifier: 126 species, 16,915 training images, RF-DETR Medium architecture. Fork it into your own account to use as-is or fine-tune for your site.

OPTION B
Roboflow Universe (other ecosystems)

Browse Roboflow Universe for a pre-trained model suited to your target species or habitat type. Fork a workspace and note the model endpoint string (format: workspace-slug/model-id/version).

OPTION C
Custom-trained model

Upload and annotate your own image dataset in Roboflow, then train a model using Roboflow's training infrastructure. Choose an architecture appropriate for your detection requirements and deployment environment.

3

Import the Roboflow Workflow template

Locate the workflow template file in the roboflow-workflows/ directory of the cloned repository. In your Roboflow workspace, navigate to Workflows and import the template. The workflow contains five pre-configured components:

  • Object detection block: runs your chosen model against each video frame and generates detection predictions
  • Detection_Converter block: converts raw detection predictions into structured data ready for downstream processing
  • Add_Webcam_Interface block: generates the real-time video overlay with live species identification ticker displayed on the output stream
  • Anthropic_Environmental_Analyzer block: sends frame detections to Claude 3.5 Sonnet at configurable intervals for preliminary environmental analysis
  • Analyst_Caller block: forwards structured detection data to your N8N RTSP Analyst webhook for deep multi-agent analysis

Repository location: roboflow-workflows/

NOTE

All workflow logic is pre-configured in the template. You only need to enter API keys and the N8N webhook URL into the Python blocks. No code editing is required.

4

Configure the Python block API keys

Open each Python block in the workflow and enter the required credentials:

  • Anthropic_Environmental_Analyzer: set ANTHROPIC_API_KEY to your Anthropic API key
  • Analyst_Caller: set webhook_url to your N8N RTSP Analyst webhook URL, obtained in Section 02, Step 5
WARN

The Analyst_Caller block requires the N8N webhook URL from Section 02. Complete Section 02 Steps 1–5 first, then return here to finish this step.

5

Record your Roboflow API key and model endpoint

Navigate to Settings → Roboflow API Key and copy your key. Also note your model endpoint string from the model deployment page (format: workspace-slug/model-id/version). Both values are referenced during N8N credential configuration in Section 02.

6

Connect your video source

Configure the video input node in the workflow to point to your source. Supported formats:

  • RTSP stream (e.g., rtsp://192.168.1.100:554/stream)
  • MP4 or video file path
  • USB or webcam input

If your camera does not transmit GPS coordinates with the video stream, note your study site latitude and longitude for manual entry in N8N (Section 02, Step 7).

7

Verify detections are generated

Run the workflow against your video source and confirm that detection events appear in the Roboflow Workflow output panel. Full end-to-end verification (detections reaching Airtable) is completed after N8N is configured and activated in Section 02.

02

N8N — RTSP Analyst Workflow

9 AI agents for deep ecological analysis, generating 72 data points per observation

The RTSP Analyst Workflow receives detection data from Roboflow via webhook and runs 9 specialized AI agents. Agents cover taxonomy and phenology, species recurrence memory, GIS validation, biodiversity intelligence, environmental status, marine climate, terrestrial climate, habitat assessment, and cartographic mapping. Results are written to two Airtable tables: Species Observations (62 columns) and Environmental Mapping (10 columns).

1

Create an N8N account or deploy self-hosted

Sign up at n8n.io for N8N Cloud (free trial available), or deploy N8N on your own server or VPS. The workflow template is compatible with both deployment modes.

2

Import the workflow template

Download RTSP_Analyst_Workflow__GitHub_Template_.json from the n8n-workflows/rtsp-analyst/ directory of the repository. In N8N, navigate to Workflows → Import and upload the file.

INFO

Repository location: n8n-workflows/rtsp-analyst/

3

Configure credentials

Navigate to Settings → Credentials → Add Credential in N8N and add the following:

Credential Type Used by
OpenAI API (or compatible LLM) API key (Authorization: Bearer) All 9 AI agents
Airtable Personal Access Token All Airtable write nodes, Memory Manager agent
IUCN Red List API key (request header) Biodiversity Intelligence agent
API Ninjas API key (request header: X-Api-Key) Systematics Phenologist, Biodiversity Intelligence agents
Mapbox Access token (query parameter: access_token) Cartography Mapper agent
NOTE

GBIF, iNaturalist, OpenStreetMap Nominatim, Open-Meteo (all endpoints), OpenTopoData, Open Elevation, NOAA Co-OPS, and SOILGRIDS require no credentials in N8N.

4

Set Airtable identifiers

In the workflow configuration nodes, enter your Base ID and Table IDs. These values are also referenced as environment variables. Set them in N8N or in a .env file:

ENV VARS 
AIRTABLE_BASE_ID=appXXXXXXXXXXXXXX
AIRTABLE_SPECIES_TABLE_ID=tblXXXXXXXXXXXXXX
AIRTABLE_MAPPING_TABLE_ID=tblXXXXXXXXXXXXXX
IUCN_API_KEY=your_iucn_key
API_NINJAS_KEY=your_api_ninjas_key
MAPBOX_ACCESS_TOKEN=your_mapbox_token

Replace all placeholder values with your actual credentials obtained from Section 04 (Airtable IDs) and the credentials table above (API keys).

5

Copy the webhook URL

Open the Webhook trigger node in the workflow. Copy the webhook URL shown. It follows this format:

WEBHOOK URL FORMAT 
https://your-n8n-instance.app.n8n.cloud/webhook/rtsp-analyst

Return to Section 01, Step 4 and paste this URL into the webhook_url variable of the Roboflow Analyst_Caller Python block.

6

Configure your ecosystem profile

The RTSP Analyst includes habitat-specific climate agents. Enable or disable them based on your study environment to avoid processing irrelevant data and conserve API token consumption:

Study environment Marine Climate Search Terrestrial Climate Search Cartography Mapper
Marine Enable Disable Enable when satellite imagery needed
Terrestrial Disable Enable Enable when satellite imagery needed
Estuarine or mixed Enable Enable Enable when satellite imagery needed
NOTE

The Cartography Mapper agent generates Mapbox satellite imagery at three zoom levels and queries NOAA for coastal marine data. Each execution calls a vision-capable model to interpret the imagery. Enable this agent only when spatial satellite data is needed for your research to keep token usage proportional to output value.

7

Set static location (cameras without GPS)

If your camera or recording device does not transmit GPS coordinates with the video stream, open the Set Static Location node in the Static Location (Optional) segment on the left side of the workflow canvas. Enter the latitude and longitude of your study site. All observations from this deployment will use these coordinates.

8

Test the webhook

Send a test detection payload to verify the workflow processes data end-to-end before activating:

CURL 
curl -X POST https://your-n8n-instance.app.n8n.cloud/webhook/rtsp-analyst \
  -H "Content-Type: application/json" \
  -d '{
    "species_name": "Test Species",
    "scientific_name": "Genus species",
    "confidence": 0.94,
    "timestamp": "2025-01-01T14:32:15Z",
    "location": {
      "latitude": 10.234,
      "longitude": -84.567
    }
  }'

Expected response: {"success": true, "observation_id": "...", "data_points": 72, "agents_executed": 9}

Verify a record appears in your Airtable Species Observations table. If the Cartography Mapper is active, a corresponding record should also appear in the Environmental Mapping table.

9

Activate the workflow

Toggle the Active switch in the top-right corner of the workflow editor. The workflow will now receive incoming detection webhooks from Roboflow. Monitor the Executions tab for activity logs and error details.

03

N8N — Daily Reporter Workflow

5 AI agents synthesizing observation data into scheduled PDF intelligence reports

The Daily Reporter Workflow runs on a configurable schedule (default: midnight UTC) and reads all observations from the preceding reporting period to generate a structured PDF report. Five agents handle species biodiversity analysis, environmental and climate assessment, habitat and conservation evaluation, cartographic and spatial analysis using vision, and final report synthesis. Reports are saved to the Airtable Daily Reports table and are downloadable from the dashboard.

1

Import the workflow template

Download Daily_Reporter__GitHub_Template_.json from the n8n-workflows/daily-reporter/ directory of the repository. In N8N, navigate to Workflows → Import and upload the file.

INFO

Repository location: n8n-workflows/daily-reporter/

2

Configure credentials

Add or reuse the following credentials in N8N (Settings → Credentials). The AI Model and Airtable credentials can be shared with the RTSP Analyst workflow:

Credential Type Used by
OpenAI API (or compatible LLM) API key All 5 AI agents
Airtable Personal Access Token Data fetch nodes, report save node
PDFShift API key PDF generation node

Obtain a PDFShift API key at pdfshift.io. The free tier supports up to 50 PDF conversions per month.

WARN

PDFShift is required for PDF generation. If the PDFShift credential is missing or invalid, the report synthesis step will complete but no PDF file will be produced and the Airtable record will not be saved.

3

Set Airtable identifiers

Configure all four Airtable identifiers required by this workflow:

ENV VARS 
AIRTABLE_BASE_ID=appXXXXXXXXXXXXXX
AIRTABLE_SPECIES_TABLE_ID=tblXXXXXXXXXXXXXX
AIRTABLE_MAPPING_TABLE_ID=tblXXXXXXXXXXXXXX
AIRTABLE_REPORTS_TABLE_ID=tblXXXXXXXXXXXXXX
4

Configure the report schedule

Open the Daily Trigger node in the workflow and set the cron expression for your desired reporting frequency. The default is midnight UTC:

CRON EXPRESSIONS 
0 0 * * *      # Daily at midnight UTC (default)
0 6 * * *      # Daily at 06:00 UTC
0 */6 * * *    # Every 6 hours
0 0 * * 1      # Weekly, Monday midnight UTC
5

Test the workflow manually

Click Execute Workflow in the workflow editor. Monitor the execution in the Executions tab. Verify that a record appears in the Airtable Daily Reports table with a populated Report PDF URL. Expected execution time is 2–5 minutes depending on observation volume.

NOTE

If no observations exist for the reporting period, the workflow generates an empty-period notification rather than a PDF report. Ensure the RTSP Analyst workflow has processed at least one observation and that records appear in the Species Observations table before running this test.

6

Activate the workflow

Toggle the Active switch in the workflow editor. The workflow will run automatically on the configured schedule.

Agent 2: Environmental & Climate Analyst Marine/terrestrial climate trends, sea surface temperature, precipitation, atmospheric conditions, oceanographic patterns, anomaly detection, seasonal context
Agent 3: Habitat & Conservation Analyst Habitat type distribution, ecosystem health scores, conservation status, degradation indicators, management recommendations, priority conservation areas

04

Airtable

Research database hosting species observations, geospatial mapping records, and generated reports

Airtable is the central database for all Audtheia data. The N8N RTSP Analyst and Daily Reporter workflows write to it; the Audtheia dashboard reads from it. Three tables are required. Example CSV files and full schema documentation are available in the airtable-schemas/ directory of the repository.

1

Create an Airtable account

Sign up at airtable.com. The free tier supports the full Audtheia schema.

2

Create the base

Create a new base and name it Environmental Data Management. This name is referenced in the Personal Access Token access configuration.

3

Create a Personal Access Token

Navigate to Builder Hub → Personal access tokens and create a new token. Configure the following before saving:

  • Scopes: add data.records:read (required for the Audtheia dashboard) and data.records:write (required for N8N to write observation data)

A missing data.records:read scope is the most common cause of "Invalid permissions" errors in the dashboard. A base that is not added to the token's access list produces the same error even when scopes are correctly set.

05

Dashboard Connection

Connect the Audtheia researcher dashboard to your Airtable base

The Audtheia dashboard connects directly to your Airtable base via the Airtable REST API using your Personal Access Token. Credentials are stored only in your browser's localStorage on your own device and are never transmitted to Audtheia servers. No Audtheia account is required to use the dashboard.

1

Open the dashboard

Navigate to the Audtheia dashboard. The settings panel is visible by default on first visit before credentials are saved.

2

Enter credentials in Settings

Fill in all five fields in the Settings section:

  • Airtable Personal Access Token: the token created in Section 04, Step 6
  • Airtable Base ID: starts with app
  • Species Observations Table ID: starts with tbl
  • Environmental Mapping Table ID: starts with tbl
  • Daily Reports Table ID: starts with tbl

Click Save Credentials. Values are written to localStorage on your device only.

3

Verify each tab loads data

Open each of the three dashboard tabs and confirm data appears:

  • Species Observations: collapsible detection records with taxonomy, rarity scores, conservation status, and full 62-field expansion
  • Environmental Mapping: satellite imagery at three zoom levels, center coordinates, and mapping metadata per observation
  • Daily Reports: report records with name, date, and PDF download link
NOTE

Tabs are lazy-loaded on first click. If a tab shows no data, wait 2–3 seconds and click again. An "Invalid permissions" error indicates a token scope or base access issue. See Section 04, Step 6 for resolution.

4

Security note

Your Airtable Personal Access Token is transmitted only to api.airtable.com as a Bearer header when the dashboard fetches your data. It is not logged or stored by Audtheia. Do not use the dashboard on shared or public computers.

Deployment complete

With all five sections complete, your Audtheia instance is fully operational. Roboflow processes your video stream; the RTSP Analyst writes 72-point observations to Airtable; the Daily Reporter generates PDF reports on schedule; and the dashboard displays your research data. If you encounter issues, see the Troubleshooting section below or open a GitHub issue.

Troubleshooting

Check the following in order:

  1. Confirm the N8N RTSP Analyst workflow is set to Active (toggle in the top-right of the workflow editor).
  2. Verify the webhook URL in the Roboflow Analyst_Caller Python block matches the URL shown in the N8N Webhook trigger node exactly, including the protocol and path.
  3. Open the N8N Executions tab. If requests are arriving but failing, the error details appear there at the node level.
  4. Run the test curl command from Section 02, Step 8 to isolate whether the problem is in Roboflow's dispatch or in N8N's processing.

This error is returned by the Airtable API when the Personal Access Token is missing a required scope or does not have access to the base. Both conditions produce the same error message.

  1. Navigate to Builder Hub → Personal access tokens and edit your token.
  2. Verify that both data.records:read and data.records:write are listed under Scopes.
  3. Under Access, confirm that your Environmental Data Management base is included.
  4. Save the token. If you regenerated it, re-enter the new value in the dashboard Settings section.

Satellite imagery is generated by the Cartography Mapper agent. If Environmental Mapping records exist but URL fields are empty:

  1. Confirm the Cartography Mapper agent is enabled in the RTSP Analyst workflow (Section 02, Step 6).
  2. Verify your Mapbox access token is correctly entered in N8N credentials. An invalid token does not fail immediately but produces empty URL fields in the output.
  3. Check your Mapbox account at account.mapbox.com for token status and monthly quota usage.

If credentials are accepted without an error but tabs display an empty state:

  1. Verify the Base ID begins with app and each Table ID begins with tbl. Pasting the wrong segment from the URL is the most common source of this issue.
  2. Confirm the RTSP Analyst workflow has processed at least one observation successfully. Tables must contain records before the dashboard can display anything.
  3. Check that table names in Airtable match exactly: Species Observations, Environmental Mapping, Daily Reports.
  1. Verify the PDFShift API key is correctly configured in N8N credentials under Settings → Credentials.
  2. Confirm observations exist in the reporting period. Run the RTSP Analyst workflow first and confirm records appear in Airtable before triggering the Daily Reporter.
  3. Verify the Daily Reports table exists with column names exactly as specified in Section 04, Step 5: Report Name, Report PDF URL, Date Uploaded.
  4. Check the N8N Executions tab for the Daily Reporter workflow for node-level error details.

Partial records indicate one or more N8N agents failed mid-execution. Check the N8N Executions tab and look for errors in specific agent nodes.

  1. IUCN or API Ninjas API key missing or rate-limited: verify credentials and account quota at their respective dashboards.
  2. Agent execution timeout: increase the timeout setting in N8N Settings (default: 120 seconds). Reduce max_tokens in the agent configuration if needed.
  3. Airtable Personal Access Token missing data.records:write scope: verify in Builder Hub.
  4. Column name mismatch in Airtable: check that all 62 Species Observations column names match the schema exactly, including capitalization and spacing.

Expected behavior. The Audtheia demo proxy is deployed on Render.com's free tier, which suspends services after 15 minutes of inactivity. The first request after a suspension period wakes the service, which takes 30–50 seconds. Subsequent requests within the same active window respond within 1–2 seconds. This only affects the public demo at demo.html and has no impact on your own Roboflow or N8N deployment.
