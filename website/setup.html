Setup Guide

Set Up Your Own Audtheia Instance

This guide walks you through setting up Audtheia from scratch. No programming experience is required. By the end, your camera feed will be detecting species automatically, your data will be flowing into a structured database you own, and daily reports will be generating on a schedule. Follow the sections in order, and take your time with each step.

01 Roboflow Detects species in your video feed at 60 FPS
02 N8N RTSP Analyst 9 AI agents analyze each detection and record 72 data points
03 N8N Daily Reporter 5 AI agents compile daily PDF research reports on a schedule
04 Airtable Your database stores all observations and reports
05 Dashboard Your researcher interface, connected to your own Airtable

Before you begin

A few things to gather before starting. Everything here is free or has a free tier. None of this requires programming experience to set up.

What you need physically

A camera that streams video, either a webcam, an IP camera, or any camera compatible with Roboflow. A regular video file also works for testing.
The GPS coordinates of your study site. Your camera may provide these automatically. If not, you can enter them manually in N8N.
Git installed on your computer. This lets you download the Audtheia template files.

Download the Audtheia repository first

The workflow templates, database schemas, and configuration files you need throughout this guide are all in the Audtheia GitHub repository. Download it now before moving on.

BASH 
git clone https://github.com/AudtheiaOfficial/audtheia-environmental-monitoring.git
cd audtheia-environmental-monitoring

API keys and credentials you will need

You will collect these one by one as you work through each section. The table below is a complete reference so nothing gets missed.

01

Roboflow

This is where your camera feed gets processed and species detections happen

Roboflow watches your video stream and identifies species in real time. When a detection happens, four pre-built code blocks inside the workflow handle the rest: drawing the live overlay on your video, converting the raw detection into structured data, running a quick AI environmental note, and sending the detection to your N8N analysis pipeline. You only need to enter your API keys, nothing else.

1

Create a free Roboflow account

Sign up at roboflow.com. The free tier covers everything Audtheia needs.

2

Choose a detection model for your species

Pick whichever option matches your field site:

OPTION A
Marine sponge research (start here)

Fork the official Porifera classifier workspace official-porifera-classifier-ju8er on Roboflow Universe. This is the reference implementation: 126 species, 16,915 training images, trained with RF-DETR Medium. Fork it into your own account and use it as-is, or fine-tune it further on your specific site's data.

OPTION B
Other ecosystems or species

Browse Roboflow Universe to find a pre-trained model for your target species. Fork a workspace and note the model endpoint string (format: workspace-slug/model-id/version).

OPTION C
Train your own custom model

Upload and label your own images in Roboflow, then train a model using Roboflow's training tools. This gives you a model tailored to your exact species and environmental conditions.

3

Import the Roboflow Workflow template

Find the workflow template file in the roboflow-workflows/ folder of the repository you cloned. In your Roboflow workspace, go to Workflows and import the file. The workflow comes with five pre-configured components already in place:

  • Object detection block runs your chosen model against each video frame and produces detection results
  • Detection_Converter converts the raw detection output into clean structured data
  • Add_Webcam_Interface generates the live species identification overlay on your video feed
  • Anthropic_Environmental_Analyzer sends detection data to an AI language model at regular intervals for a quick environmental note
  • Analyst_Caller forwards the structured detection to your N8N analysis workflow via a web address (webhook)

Repository location: roboflow-workflows/

NOTE

All the logic is already built into the template. You only need to fill in your API keys. No coding required.

4

Add your API keys to the workflow blocks

Open each Python block in the workflow and enter the values listed below:

  • Anthropic_Environmental_Analyzer: set ANTHROPIC_API_KEY to your Anthropic API key
  • Analyst_Caller: set webhook_url to the N8N webhook URL you will get in Section 02, Step 5
WARN

You need the N8N webhook URL from Section 02 before you can finish this step. Complete Section 02 Steps 1 through 5, then come back here.

5

Note down your Roboflow API key and model endpoint

Go to Settings → Roboflow API Key and copy your key. Also note the model endpoint string from your model's deployment page. Both values are needed when you set up N8N in Section 02.

6

Connect your video source

Point the video input in the workflow at your camera or recording. Supported formats:

  • RTSP stream from an IP camera (e.g., rtsp://192.168.1.100:554/stream)
  • An MP4 or other video file for testing
  • A USB webcam connected to your computer

If your camera doesn't automatically transmit GPS coordinates, note your site's latitude and longitude for manual entry in N8N (Section 02, Step 7).

7

Run the workflow and verify detections appear

Start the workflow and confirm that species detections show up in the Roboflow output panel. Full end-to-end testing (confirming detections arrive in Airtable) happens after you complete Section 02.

02

N8N — RTSP Analyst Workflow

Nine AI agents analyze every detection and write 72 structured data points to your database

Every time Roboflow detects a species, it sends a message to N8N. Nine specialized AI agents then go to work: one identifies the full taxonomy, another checks whether this individual has been seen before, a third validates the GPS location, others look up the conservation status, pull real-time environmental conditions, assess the habitat, and generate a satellite map. The entire output, 72 structured data points, gets written to your Airtable database.

1

Create an N8N account or host it yourself

Sign up at n8n.io for a cloud account (free trial available), or deploy N8N on your own server. Both work with the Audtheia workflow template.

2

Import the RTSP Analyst workflow

Find RTSP_Analyst_Workflow__GitHub_Template_.json in the n8n-workflows/rtsp-analyst/ folder of the repository. In N8N, go to Workflows → Import and upload the file.

INFO

Repository location: n8n-workflows/rtsp-analyst/

3

Add your credentials to N8N

Go to Settings → Credentials → Add Credential and add the following. These are stored securely inside N8N and never leave your instance:

Credential Type Used by
OpenAI API (or compatible LLM) API key (Authorization: Bearer) All 9 AI agents
Airtable Personal Access Token All Airtable write nodes, Memory Manager agent
IUCN Red List API key (request header) Biodiversity Intelligence agent
API Ninjas API key (request header: X-Api-Key) Systematics Phenologist, Biodiversity Intelligence agents
Mapbox Access token (query parameter: access_token) Cartography Mapper agent
NOTE

GBIF, iNaturalist, OpenStreetMap Nominatim, all Open-Meteo endpoints, OpenTopoData, Open Elevation, NOAA Co-OPS, and SOILGRIDS are completely free and require no credentials. They work automatically.

4

Enter your Airtable database identifiers

In the workflow's configuration nodes, enter your Airtable Base ID and Table IDs. You can also set them as environment variables:

ENV VARS 
AIRTABLE_BASE_ID=appXXXXXXXXXXXXXX
AIRTABLE_SPECIES_TABLE_ID=tblXXXXXXXXXXXXXX
AIRTABLE_MAPPING_TABLE_ID=tblXXXXXXXXXXXXXX
IUCN_API_KEY=your_iucn_key
API_NINJAS_KEY=your_api_ninjas_key
MAPBOX_ACCESS_TOKEN=your_mapbox_token

You will get the Base ID and Table IDs when you set up Airtable in Section 04. Come back here to fill these in after that.

5

Copy the webhook URL

Open the Webhook trigger node in the workflow and copy the URL it shows. It will look something like this:

WEBHOOK URL FORMAT 
https://your-n8n-instance.app.n8n.cloud/webhook/rtsp-analyst

Go back to Section 01, Step 4 and paste this URL into the webhook_url field in the Roboflow Analyst_Caller block.

6

Choose which climate agents to enable for your habitat

The RTSP Analyst has separate agents for marine and terrestrial environments. Enable only the ones that match your study site to keep things efficient:

Your study environment Marine Climate Search Terrestrial Climate Search Cartography Mapper
Marine Enable Disable Enable when satellite imagery needed
Terrestrial Disable Enable Enable when satellite imagery needed
Estuarine or mixed Enable Enable Enable when satellite imagery needed
NOTE

The Cartography Mapper generates Mapbox satellite images at three zoom levels and uses a vision-capable AI model to interpret them. It is optional. Enable it only when spatial satellite imagery is genuinely useful for your research.

7

Set your site location (if your camera doesn't transmit GPS)

If your camera or recording device doesn't automatically include GPS coordinates, open the Set Static Location node on the left side of the workflow canvas and enter your site's latitude and longitude. All observations will use these coordinates.

8

Send a test detection to check everything is connected

Run this command in your terminal to send a test detection through the workflow:

CURL 
curl -X POST https://your-n8n-instance.app.n8n.cloud/webhook/rtsp-analyst \
  -H "Content-Type: application/json" \
  -d '{
    "species_name": "Test Species",
    "scientific_name": "Genus species",
    "confidence": 0.94,
    "timestamp": "2025-01-01T14:32:15Z",
    "location": {
      "latitude": 10.234,
      "longitude": -84.567
    }
  }'

A successful response looks like: {"success": true, "observation_id": "...", "data_points": 72, "agents_executed": 9}

Then check your Airtable database. A new record should appear in the Species Observations table. If the Cartography Mapper is enabled, a record should also appear in the Environmental Mapping table.

9

Activate the workflow

Toggle the Active switch in the top-right corner of the N8N workflow editor. The workflow will now receive live detections from Roboflow automatically. Check the Executions tab at any time to see what's been processed and any error details.

What the nine agents do

Agent Data sources What it records
Systematics Phenologist GBIF, iNaturalist, API Ninjas Full taxonomy, seasonal timing, behavioral patterns (15 data points)
Memory Manager Your Airtable records Previous sightings of this individual, pattern changes, anomalies (5 data points)
GIS Data Manager OpenStreetMap, OpenTopoData, Open Elevation Confirmed GPS location, elevation, biogeographic region (8 data points)
Biodiversity Intelligence GBIF, IUCN Red List, iNaturalist, API Ninjas Conservation status, rarity score, ecological role (9 data points)
EnviroStatus Manager Open-Meteo (4 APIs) Habitat type, ecosystem classification, environmental suitability (6 data points)
Marine Climate Search NOAA Co-OPS, Open-Meteo Marine, Open Elevation Sea surface temperature, currents, wave height, salinity (8 data points, marine only)
Terrestrial Climate Search Open-Meteo Weather, SOILGRIDS, Open Elevation Temperature, rainfall, humidity, soil type, biome (8 data points, terrestrial only)
Habitat Assessment NDVI, Open Elevation, Open-Meteo Vegetation index, ecosystem health, habitat quality, degradation indicators (7 data points)
Cartography Mapper Mapbox Static Images API, NOAA Satellite imagery at three zoom levels, bathymetric data (10 data points, optional)

Full documentation: n8n-workflows/rtsp-analyst/README.md

03

N8N — Daily Reporter Workflow

Five agents compile your observations into a PDF research report on a schedule you set

The Daily Reporter runs automatically on a schedule. Five agents read all the observations from the reporting period and each takes on a different analytical section of the report: species diversity, environmental and climate patterns, habitat health, satellite mapping, and a full FAIR-compliant summary. The finished PDF is saved to your Airtable Daily Reports table and downloadable from the researcher dashboard.

1

Import the Daily Reporter workflow

Find Daily_Reporter__GitHub_Template_.json in the n8n-workflows/daily-reporter/ folder. In N8N, go to Workflows → Import and upload the file.

INFO

Repository location: n8n-workflows/daily-reporter/

2

Add or reuse credentials

This workflow uses three credentials. If you already added the AI model and Airtable credentials in Section 02, you can reuse them here:

Credential Type Used by
OpenAI API (or compatible LLM) API key All 5 AI agents
Airtable Personal Access Token Data retrieval and report saving
PDFShift API key PDF generation

Get your PDFShift API key at pdfshift.io. The free tier allows up to 50 PDF conversions per month.

WARN

PDFShift is required for report generation. Without a valid PDFShift key, the workflow will complete its analysis but no PDF will be saved and no Airtable record will be created.

3

Enter your Airtable identifiers

Configure the four Airtable values this workflow needs:

ENV VARS 
AIRTABLE_BASE_ID=appXXXXXXXXXXXXXX
AIRTABLE_SPECIES_TABLE_ID=tblXXXXXXXXXXXXXX
AIRTABLE_MAPPING_TABLE_ID=tblXXXXXXXXXXXXXX
AIRTABLE_REPORTS_TABLE_ID=tblXXXXXXXXXXXXXX
4

Set the report schedule

Open the Daily Trigger node and enter a cron expression for how often you want reports generated. The default is midnight UTC every day:

CRON EXAMPLES 
0 0 * * *      # Every day at midnight UTC (default)
0 6 * * *      # Every day at 06:00 UTC
0 */6 * * *    # Every 6 hours
0 0 * * 1      # Every Monday at midnight UTC
5

Run a manual test

Click Execute Workflow to trigger a report now. Check the Executions tab as it runs. When finished, open your Airtable Daily Reports table and confirm a new record appeared with a PDF URL. Reports typically take 2 to 5 minutes depending on how many observations exist.

NOTE

If no observations exist yet for the reporting period, the workflow produces an empty-period notification rather than a full PDF report. Make sure the RTSP Analyst has processed at least one observation first.

6

Activate the workflow

Toggle the Active switch in the workflow editor. Reports will now generate automatically on your chosen schedule.

What the five agents write in each report

Agent Report section
Agent 1: Species and Biodiversity Analyst Species diversity indices, conservation status breakdown, rare and endemic species, ecological roles, activity patterns over the period
Agent 2: Environmental and Climate Analyst Climate trends, sea surface temperature, precipitation, atmospheric conditions, oceanographic patterns, seasonal context, anomaly flags
Agent 3: Habitat and Conservation Analyst Habitat type distribution, ecosystem health scores, threat indicators, degradation, management priority recommendations
Agent 4: Cartography and Spatial Analyst (uses vision) Interprets satellite imagery across three zoom levels, geographic clustering, habitat connectivity, bathymetric analysis, land cover
Agent 5: Report Synthesis and Generation Combines all sections into a structured narrative, assembles a 12-section HTML report, documents FAIR compliance, generates the final PDF via PDFShift

Full documentation: n8n-workflows/daily-reporter/README.md

04

Airtable

Your research database, storing every observation, map, and report

Airtable is the database that holds all of your observation data. The N8N workflows write to it and the Audtheia dashboard reads from it. You need to create three tables. The fastest way is to import the example CSV files from the repository, which creates the right column structure automatically. Column names must match exactly as listed, because the N8N workflows reference them by name.

Schema documentation and example CSV files are in: airtable-schemas/

1

Create a free Airtable account

Sign up at airtable.com. The free tier fully supports the Audtheia database schema.

2

Create a new base named "Environmental Data Management"

This exact name is referenced during Personal Access Token setup. Inside this base you will create the three tables below.

3

6

Create a Personal Access Token

Go to Builder Hub → Personal access tokens and create a new token. Before saving, configure the following:

  • Scopes: add data.records:read (for the dashboard) and data.records:write (for N8N to write observations)
  • Access: add your Environmental Data Management base to the token's access list
WARN

Copy the token immediately after creation. Airtable shows it only once. If you lose it, you must generate a new one and update all N8N workflows and the dashboard.

NOTE

A missing data.records:read scope is the most common cause of "Invalid permissions" errors in the dashboard. Adding the base to the token's access list is also required, even when the scopes are correct.

7

Note down your Base ID and Table IDs

These appear in the Airtable URL when you open your base and each table. You need one Base ID and three Table IDs:

                         ^^^^^^^^^^^^^^^^

# Table ID appears when a specific table is open (starts with "tbl")
https://airtable.com/appXXXXXXXXXXXXXX/tblXXXXXXXXXXXXXX/...
                                         ^^^^^^^^^^^^^^^^

Record these four values. Enter them into N8N (Sections 02 and 03) and into the dashboard settings (Section 05).

05

Dashboard Connection

Connect the Audtheia researcher dashboard to your Airtable base

The dashboard reads directly from your Airtable base using your own credentials. You enter your token and base identifiers once in the dashboard settings, and they are saved locally in your browser. Nothing is sent to Audtheia. No Audtheia account is needed.

1

Open the dashboard

Go to the Audtheia dashboard. The settings panel appears automatically on the first visit.

2

Enter your Airtable credentials in Settings

Fill in all five fields:

  • Airtable Personal Access Token: the token you created in Section 04, Step 6
  • Airtable Base ID: starts with app
  • Species Observations Table ID: starts with tbl
  • Environmental Mapping Table ID: starts with tbl
  • Daily Reports Table ID: starts with tbl
3

Save and verify the connection

Click Save Credentials. The dashboard will attempt to connect to your Airtable base. If credentials are correct and observations exist, the Species Observations tab will populate with records. If the table is empty, that is expected until the RTSP Analyst has processed at least one detection.

Troubleshooting

If something isn't working, these are the most common causes. Check the N8N Executions tab first, it shows exactly where a failure occurred and what the error message was.

  1. Confirm the N8N RTSP Analyst workflow is Active (toggle is on in the top-right corner of the workflow editor).
  2. Confirm the webhook URL in the Roboflow Analyst_Caller Python block matches exactly what the N8N Webhook trigger node shows.
  3. Check that the Roboflow workflow is running against a live video source and producing detections in the output panel.
  1. Confirm the Personal Access Token includes both data.records:read and data.records:write scopes.
  2. Confirm the Environmental Data Management base was added to the token's access list when you created it. This is a separate step from adding scopes and is easy to miss.
  3. Check that the Base ID starts with app and all three Table IDs start with tbl. Copying the wrong URL segment is the most common mistake here.
  1. Confirm the Cartography Mapper agent is enabled in the RTSP Analyst workflow (the toggle in the agent node).
  2. Verify your Mapbox access token is correctly entered in N8N credentials. An invalid token fails silently and produces empty URL fields in the Airtable record.
  3. Check your Mapbox account at account.mapbox.com for token status and monthly usage quota.

If credentials are accepted without an error but tabs show nothing:

  1. Verify the Base ID starts with app and each Table ID starts with tbl.
  2. Confirm the RTSP Analyst workflow has processed at least one observation. Tables must contain records before the dashboard has anything to show.
  3. Check that table names in Airtable match exactly: Species Observations, Environmental Mapping, Daily Reports, including capitalization and spacing.
  1. Verify the PDFShift API key is correctly entered in N8N credentials under Settings → Credentials.
  2. Confirm observations exist for the reporting period. Run the RTSP Analyst first and verify records appear in Airtable before triggering the Daily Reporter.
  3. Verify the Daily Reports table exists with column names exactly as specified in Section 04, Step 5: Report Name, Report PDF URL, Date Uploaded.
  4. Check the N8N Executions tab for the Daily Reporter workflow to find specific node errors.

Partial records mean one or more N8N agents failed mid-run. Check the N8N Executions tab for errors on specific agent nodes.

  1. IUCN or API Ninjas key missing or rate-limited: verify credentials and account quota.
  2. Agent execution timeout: increase the timeout setting in N8N Settings. Reducing max_tokens in the agent configuration can also help.
  3. Airtable Personal Access Token missing the data.records:write scope: verify in Builder Hub.
  4. Column name mismatch in Airtable: confirm all 62 Species Observations column names match the schema exactly, including capitalization and spaces.

This is normal. The Audtheia demo proxy runs on Render.com's free tier, which pauses the service after 15 minutes without traffic. The first request wakes it up, which takes 30 to 50 seconds. After that, requests are fast. This only affects the public demo at demo.html and has no impact on your own Roboflow or N8N deployment.

Getting Help

If you run into something not covered above, open a GitHub issue. Include the N8N execution log, the error message from Airtable or the dashboard, and the section and step number where things stopped working. This helps narrow it down quickly.

The repository also has full workflow documentation, Airtable schema files, and annotated example CSV files that cover edge cases not addressed in this guide.
